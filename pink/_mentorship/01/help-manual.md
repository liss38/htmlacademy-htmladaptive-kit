			<!-- здесь разметим футер сайта -->

##### На что следует обратить внимание:

`<meta content="width=device-width, minimum-scale=1.0, maximum-scale=1.0" name="viewport">` - это строка пригодится в будущем,
запоминать её сейчас не нужно, если по сути, то благодаря ей сайты становятся _действительно_ адаптивными на разных устройствах,
она отменяет дефолтное масштабирование в мобильных браузерах

`<title>Pink - конкурс на самое безбашенное путешествие с Pink</title>` - для каждой страницы задан свой "тайтл"

`<meta charset="UTF-8">` - задана кодировка utf8

`<link rel="stylesheet" href="css/style.min.css">` - сразу подключен пока еще несуществующий стилевой файл проекта

`page`, `page__header`, `page__content`, `page__footer` - задано пространство имен для css-классов согласно БЭМ для базовой сетки страницы,
где `page` - служит оберткой для всей страницы целиком(холст) и это **БЭМ-блок**,
а `page__header`, `page__content`, `page__footer` - обертки для шапки, контентной части и футера сайта и они являются 
**БЭМ-элементами БЭМ-блока** `page`



# 5. LESS__ORDER: структура less-каталога
Поскольку:

 1. Согласно критерию **Б13** для каждого БЭМ-блока должен быть создан отдельный less-файл, который называется по имени БЭМ-блока(например page.less, navi.less, button.less, ...) стили которого он содержит,
 2. а таких блоков/файлов может оказаться очень-очень-очень много и каталог less будет похож на помойку, то
 3. чтобы избежать _выноса мозга_ в конце работы при поиске и подключении нужного less-файла
 4. следует саму папку `less` разбить на подкаталоги:

```
	less
	----
		mixins
		layout
			fonts
			external
			grid
			uikit
			header
			content
			footer

		pages
			index
			photo
			form

		system

```


less/
=
1. **mixins** - здесь находятся less-файлы с миксинами и переменными, которые используются в проекте
2. **layout** - здесь находятся less-файлы с базовыми стилями, которые есть на всех страницах проекта(из которых можно построить основу для любой страницы текущего проекта)
3. **pages** - здесь находятся les-файлы со стилями страниц проекта
4. **system** - здесь находятся системные/отладочные les-файлы, папка необязательная и на продакшн не идет


less/layout/
=
1. **fonts** - кроссбраузерное подключение шрифтов в less-файле
2. **external** - less-файлы внешних библиотек(normalize.less, reset.less, less-файли для jquery-плагинов), может не понадобится в этом проекте
3. **grid** - базовая сетка проекта
4. **uikit** - типовые элементы интерфейса: кнопки, пагинаторы, ссылки, элементы форм, ...
5. **header** - стили для шапки сайта(меню, лого и т.д.)
6. **content** - стили для контента, эта папка скорее всего не понадобится
7. **footer** - стили футера


less/pages/
=
1. **index** - стили главной страницы
2. **photo** - стили страницы с пользовательскими фото
3. **form** - стили страницы с формой



# 6. style.less: подготовительное оформление файла
Файл style.less находится в корне каталога less/ и никаких стилей не содержит, в нем с помощью команды `@import` подключаются все остальные less-файлы, т.е. style.less состоит только из "импортов", переносов строк и комментариев, он выполняет роль "объединятеля" отдельных less-файлов(button.less, pagination.less, header.less, ...) в одну "команду", из которой потом с пмощью Gulp/Grunt формируется файл style.min.css

**style.less**

```
/* 
 **********
 * MIXINS *
 **********
 */
@import "mixins/fonts.less"; // примеси для шрифтов
@import "mixins/adaptive.less"; // брекпоинты для адаптива
@import "mixins/images.less"; // пути для картинок и примеси для спрайтов




/* 
 **********
 * LAYOUT *
 **********
 */




/* 
 *********
 * PAGES *
 *********
 */



/* 
 **********
 * SYSTEM *
 **********
 */
```
